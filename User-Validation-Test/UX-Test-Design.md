# Pyttern Visualizer UX Test Design

## 1. Introduction (15 min)

- Explain what Pyttern is.
- Show an example of a pattern file and a code file so participants understand how matching works.
- Show the structure of an example Compound pattern.

## 2. Pattern and compound task

- Give participants pre-written pattern files (sub-patterns).
- Ask them to build a compound pattern using these sub-patterns.
- Specify the requirement either in natural language (e.g. "Create a compound pattern that matches 'early_return' and 'incforloop', but does not match 'noemptyinit' or 'return'") or as a logical expression so the intent is clear.

## 3. Code files

- Provide a set of prepared code files that cover: matched, not matched, and not-validated cases.

## 4. Test execution (Compound section)

- Participants use the Pyttern Visualizer **Compound** section to complete the workflow.
- They answer task-based questions in a form (e.g. Google Forms) as they go, so they follow the tasks without being told step-by-step instructions (e.g. "press upload").
- **Form questions:**
  - Which code files matched the compound pattern?
  - Which code files matched the two specified sub-patterns?
  - Which code files were not validated?
  - For one non-validated file, what was the issue? (short explanation)

## 5. Post-test questionnaire (15 questions)

- **SUS:** 10 standard questions.
- **Workflow-related:** 4 questions about the workflows.
- **Open-ended:** 1 open-ended question, e.g. "What would you change?"
